# بوابة المريض — توثيق نظام التصميم

## نظرة عامة

نظام تصميم مظلم احترافي مستوحى من Apple Health و Google Health.  
الخط: **Cairo** · الاتجاه: **RTL** · الثيم: **Dark Mode فقط**

## الألوان

### الخلفيات

| المتغير | القيمة | الاستخدام |
|---------|--------|-----------|
| `--color-bg-primary` | `#060912` | الخلفية الرئيسية للصفحة — أعمق طبقة |
| `--color-bg-deep`    | `#080c18` | غلاف الهاتف — طبقة وسطى |
| `--color-bg-body`    | `#0a0f1e` | محتوى الشاشة — أفتح طبقة |

> الثلاث طبقات تخلق عمقاً بصرياً دون الحاجة لحدود مرئية.

### النصوص

```css
--text-primary : #f8fafc               /* نص رئيسي — شبه أبيض */
--text-muted   : rgba(255,255,255,0.4) /* نص ثانوي — 40% شفافية */
--text-subtle  : rgba(255,255,255,0.2) /* نص خفيف جداً — عناوين الأقسام */
```

> لا تستخدم ألوان نص ثابتة — استخدم دائماً مستويات الشفافية للحصول على تناسق تلقائي مع أي خلفية داكنة.

## التدرجات اللونية

كل بطاقة خدمة تستخدم تدرجاً من اللون الداكن إلى الفاتح (145deg):

```
dark → light  (topLeft → bottomRight)
```

هذا يضمن:
1. العمق البصري — الزاوية العلوية اليسرى أغمق
2. التباين مع الأيقونة البيضاء في المنتصف
3. التوافق مع اتجاه الضوء الطبيعي (أعلى اليسار)

### تدرج بطاقة الرصيد

```css
background: linear-gradient(135deg, #0f2044 0%, #0a1a38 100%);
```

الزاوية 135deg تعطي انحدار من اليمين العلوي لليسار السفلي — مناسب لـ RTL.

## الظلال (Glows)

الظلال الملونة هي المصدر الوحيد للعمق — لا `border` مرئية:

```css
/* مثال — Blue Glow */
box-shadow:
  0 8px 28px rgba(37,99,235,0.4),        /* ظل خارجي ملون */
  inset 0 1px 0 rgba(255,255,255,0.15);  /* خط ضوء داخلي علوي */
```

الـ `inset` يحاكي انعكاس الضوء من الأعلى — تأثير زجاجي طبيعي.

## الحركات

### riseIn — دخول العناصر

```css
@keyframes riseIn {
  from { opacity: 0; transform: translateY(22px) scale(0.97); }
  to   { opacity: 1; transform: translateY(0)    scale(1); }
}
```

- **22px** ارتفاع معقول — ليس مبالغاً ولا غير ملحوظ
- **scale(0.97)** يضيف إحساس "نمو" خفيف
- المدة الموصى بها: **500ms** مع `cubic-bezier(0.34, 1.56, 0.64, 1)`

### التأخيرات المتتالية (Stagger)

```
status-bar  → 50ms
header      → 120ms
balance     → 200ms
section     → 280ms
grid item 1 → 360ms
grid item 2 → 400ms
...
grid item 6 → 560ms
cta button  → 420ms
```

الفارق 40ms بين كل عنصر شبكة — كافٍ للإحساس بالتسلسل دون إطالة.

### shimmer — بريق الزر

```css
@keyframes shimmer {
  0%   { left: -100%; }
  100% { left:  200%; }
}
/* المدة: 3s  |  infinite  |  linear */
```

**لماذا 3 ثوان؟** أسرع يبدو مزعجاً، أبطأ لا يلاحظه المستخدم.

### Spring Easing

```
cubic-bezier(0.34, 1.56, 0.64, 1)
```

النقطة `1.56` تتجاوز الـ 1.0 — هذا يخلق تأثير "ارتداد" خفيف.  
مناسب لـ hover effects وانيميشن الدخول. **لا** تستخدمه لـ exit animations.

## Glass Morphism

```css
background:        rgba(255,255,255,0.04);
border:            1px solid rgba(255,255,255,0.08);
backdrop-filter:   blur(12px);
```

> تحذير: `backdrop-filter` يكلف على GPU. استخدمه فقط على عناصر ثابتة أو قليلة الحركة. لا تضعه داخل ScrollView أو ListView.

---

## الأداء

| التقنية | لماذا |
|---------|-------|
| `transform` بدل `top/left` | يعمل على GPU thread — لا يسبب reflow |
| `will-change: transform` | يخبر المتصفح بالتحضير المسبق |
| `backdrop-filter` بحذر | مكلف جداً — استخدم فقط عند الضرورة |
| `animation-fill-mode: both` | يحافظ على حالة العنصر قبل وبعد الحركة |
